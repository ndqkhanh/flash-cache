Key Innovation

Three Protocol Stacks,
One Server

FlashCache speaks three wire protocols from a single NIO event loop: RESP (Redis-compatible) for cache operations, HTTP/2 with HPACK header compression for REST API and admin, and WebSocket for real-time cache invalidation notifications.

This demonstrates deep networking knowledge — each protocol is hand-built from ByteBuffers, not imported from a library. No Netty, no Spring, no Jackson — pure NIO from socket accept to response frame.

System Design

Architecture Overview

Every connection enters a single non-blocking Selector loop. Protocol routing happens before any application logic, so all three stacks share the same I/O thread pool.

Component Deep Dive

Every Byte Intentional

Click any component to expand implementation details. Each subsystem is hand-built with no external runtime dependencies.

O(1) LRU ConcurrentHashMap TTL Scheduler Doubly-Linked List

Core storage backed by ConcurrentHashMap for thread-safe reads without locking. LRU eviction uses a doubly-linked list + HashMap pair: get and put are O(1) with no list traversal. TTL uses a combination of lazy expiry (check-on-access) and an active scheduler that sweeps expired keys periodically, keeping memory bounded without excessive CPU overhead.

Redis Compatible Pipelining Bulk Strings Arrays

Full Redis Serialization Protocol decoder and encoder built on top of Java NIO ByteBuffers. Supports simple strings, errors, integers, bulk strings, and arrays. Command pipelining is handled natively — multiple commands in one TCP segment are processed without round-trip wait. Compatible with redis-cli, Lettuce, and Jedis clients.

HPACK Stream Multiplexing Frame Parser REST API

Hand-built HTTP/1.1 request parser for upgrade negotiation and REST fallback. HTTP/2 framing layer handles SETTINGS, HEADERS, DATA, WINDOW_UPDATE, and RST_STREAM frames. HPACK header compression implemented with static table (61 entries) and dynamic table with LRU eviction, reducing header overhead by up to 85% on repeated requests.

RFC 6455 Frame Codec Masking Ping/Pong

RFC 6455 upgrade handshake from HTTP/1.1 via Sec-WebSocket-Key / Accept header computation (SHA-1 + base64, no library). Frame codec handles FIN bit, opcode, masking key XOR, and variable-length payloads (7-bit, 16-bit, 64-bit length variants). Ping/Pong keepalive is automatic, close handshake is bidirectional and graceful.

TLS 1.3 SSLContext SNI Certificate Mgmt

TLS termination using Java's built-in SSLEngine, configured for TLSv1.3 only with strong cipher suites. SSLContext bootstrapped from JKS or PKCS#12 keystores. Non-blocking handshake integrated into the NIO Selector loop without blocking threads. Supports SNI for virtual hosting and hot certificate reloading without server restart.

NIO Selector Non-blocking I/O Event Loop Java 21

Single Selector event loop multiplexes thousands of connections on one thread. OP_ACCEPT, OP_READ, OP_WRITE interest sets managed per channel. Write buffers are queued when the socket is not ready, retried on next OP_WRITE event — zero blocking. Java 21 Virtual Threads used for handler dispatch to keep the selector loop tight while allowing handlers to do blocking work naturally.

Hash Ring Virtual Nodes Minimal Redistribution TreeMap

Hash ring implemented with a TreeMap<Long, Node> for O(log n) key lookup via ceilingKey(). Each physical node is mapped to 150+ virtual nodes to achieve uniform distribution. When a node joins or leaves, only 1/N keys are remapped on average — far superior to modulo-based sharding, which remaps nearly all keys on topology change.

SWIM-lite Failure Detection Membership Epidemic Broadcast

Simplified SWIM (Scalable Weakly-consistent Infection-style Membership) protocol for peer discovery and failure detection. Nodes periodically ping a random subset of peers; suspected nodes are indirect-pinged via k third-party nodes before marked down. Membership state propagates via piggybacking on existing messages — O(log N) dissemination time.

Async Replication Sequence Tracking Primary → Replica WAL

Async primary-to-replica replication with monotonic sequence numbers. The primary logs mutations into an in-memory write-ahead log and streams them to replicas. Replicas track their applied sequence offset; on reconnect, they request a delta replay from the primary rather than a full resync, minimising bandwidth on transient failures.

Channel Subscribe Pattern Match Keyspace Notify WS Broadcast

Pub/sub with exact channel subscriptions and glob pattern matching (PSUBSCRIBE). Keyspace notifications emit events on SET, DEL, EXPIRE, and EVICT operations, enabling cache invalidation without polling. Subscribers connected over WebSocket receive push frames in real-time, no long-polling, no SSE fallback required.

Applications

What You Can Build

FlashCache is production-capable for a wide range of distributed systems problems — anywhere low-latency shared state is required.

🔑
Session Storage

Sub-millisecond session lookups for web applications. Store user tokens with TTL; automatic expiry keeps the store clean without background jobs.

⚡
API Response Caching

Cache expensive database queries or third-party API calls with configurable TTL. HTTP/2 REST API makes integration seamless from any language.

📊
Real-Time Dashboards

WebSocket push for live cache invalidation events. Connected clients receive updates the instant a key changes — no polling, zero lag.

🚦
Rate Limiting

Atomic counter increments with TTL for API rate limiting. INCR + EXPIRE in a single pipelined RESP call — consistent across every node in the cluster.

🔒
Distributed Locking

SET with NX (not-exists) and EX (TTL) for distributed mutex. Atomic conditional set guarantees exactly-once acquisition across all clients.

📡
Pub/Sub Messaging

Real-time event notifications between microservices. Pattern subscriptions let consumers filter events by topic prefix without separate message broker infrastructure.
